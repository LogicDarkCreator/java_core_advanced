## 📝 Описание

Приложение позволяет управлять списком клиентов через консольный интерфейс.
Реализована полная валидация входных данных, обработка исключительных ситуаций
и подробное логирование всех действий и ошибок.

## ✨ Функциональность

- ✅ Добавление новых клиентов с валидацией email и телефона
- ✅ Просмотр списка всех клиентов
- ✅ Удаление клиента по имени
- ✅ Подсчет общего количества клиентов
- ✅ Справка по доступным командам
- ✅ Валидация всех входных данных
- ✅ Обработка ошибок с понятными сообщениями
- ✅ Подробное логирование в отдельные файлы
- ✅ Unit-тестирование ключевых компонентов

## 🛠 Технологии

- **Java 11** - язык программирования
- **Maven** - сборка проекта и управление зависимостями
- **Log4j2** - логирование запросов и ошибок
- **JUnit 5** - модульное тестирование
- **ConcurrentHashMap** - потокобезопасное хранение данных

## 🚀 Установка и запуск

### Предварительные требования
- JDK 11 или выше
- Maven 3.6 или выше

### Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/ConsoleCustomerList.git
cd ConsoleCustomerList
```
2. Соберите проект с помощью Maven
```bash
mvn clean compile assembly:single 
```
3. Запустите приложение
```bash
 java -jar target/ConsoleCustomerList-1.0.0-jar-with-dependencies.jar
```
Или через Maven:
```bash
mvn exec:java -Dexec.mainClass="com.customer.App"
```

| Команда	 | Описание	                        |   Пример |

| `add`	     | Добавить нового клиента	    | `add Иван Петров ivan@mail.ru +79001234567`

| `exit`	 | Выйти из программы	        | `exit`     

# ✅ Валидация данных
## Email
- Формат: username@domain.zone
- Допустимые символы: буквы, цифры, точки, дефисы, плюсы, подчеркивания
- Обязательно наличие @ и домена

## Телефон
- Формат: +71234567890 или 81234567890
- Длина: 10-15 цифр
- Допускается знак + в начале
- Пробелы, дефисы и скобки игнорируются

## Формат ввода
- Строка должна содержать ровно 4 компонента:
    - Имя
    - Фамилия
    - Email
    - Телефон

## 🚨 Обработка ошибок
Приложение обрабатывает следующие исключительные ситуации:

1. Неверное количество параметров

    - Слишком мало данных

    - Слишком много данных

2. Неверный формат email

    - Отсутствие @

    - Недопустимые символы

    - Пустой email

3. Неверный формат телефона

    - Недопустимые символы

    - Неправильная длина

    - Пустой телефон

4. Ошибки выполнения

    - Попытка удалить несуществующего клиента

    - Пустые команды

    - Неизвестные команды

## 📝 Примеры работы
Успешное добавление клиента
```text
> add Иван Петров ivan@mail.ru +79001234567
Клиент успешно добавлен: Иван Петров
```

### Ошибка валидации
```text
> add Петр Сидоров invalid-email +79001234567
Ошибка: Неверный формат email: 'invalid-email'. Ожидается формат: username@domain.zone
```

### Просмотр списка
```text
> list
Список клиентов:
Иван Петров - ivan@mail.ru - +79001234567
Всего: 1
```

### Удаление клиента
```text
> remove Иван Петров
Клиент "Иван Петров" успешно удален
```
